Regex in JavaScript

Regular expressions are used in programming languages to match strings. You create patterns to help you do that matching.

.test() method takes the regex and applies it to the string(which is placed inside the parantheses), and returns true or false if your pattern finds something or not.

You can extract matches with the .match() method. Apply the method on a string and pass in the regex inside the parantheses

You can search for multiple patters using the alternation or OR operator: |

To ignore case use the i flag /regexhere/i

To search or extract a pattern more than once, use the g flag.

The wildcard character . will match any one character. It is also called dot and period.

Character classes can be placed inside []. They allow you to define a group of characters you wish to match. Specify a range of characters using the hyphen -. Eg. /[a-z0-9]/ig;

Negated character sets. To create these, place a caret (^) after the opening bracket and before the characters you do not want to match.

Outside of a character set, the caret(^) is used to search for patterns at the beginning of strings and the dollar sign ($) at the end of the regex to search for patterns at the end of strings.

To match a character that occurs once or more in a row, use the plus (+) character. This means that it occurs atleast once and may be repeated. The character or pattern has to be present consecutively.

To match a character that occurs zero or more times, use the asterisk (*) character.

greedy match v/s lazy match. Regex are by default greedy. Use the character ? to change it to lazy matching.

Shorthand character classes: Use \w to match [A-Za-z0-9_]. This character class matches upper and lower case letters plus numbers and also includes the underscore character. To match everything but letters and numbers, use \W to match [^A-Za-z0-9_]. Shortcut to look for digit characters is \d which is equal to [0-9]. \D is equal to [^0-9]. Use shortcut \s to search for whitespace and \S to search for everything except whitespace.

Quantity specifiers: ({ }) You put two numbers between the curly brackets for the lower and upper number of patterns. These can be used with other shortcuts

Lookaheads: (There are two kinds) positive lookahead (?=...) and negative lookahead (?!...)

A positive lookahead will look to make sure the element in the search pattern is there, but wont actually match it. A negative lookahead will look to make sure the element in the search pattern is not there.

A more practical use of lookaheads is to check two or more patterns in one string.

Capture groups uses parantheses ( and ) for searching repeat substrings. You put the regex of the pattern that will repeat in between the parentheses. To specify where that repeat string will appear, you use a backslash ( \ ) and then a number. This number starts at 1 and increases with each additional capture group you use.

You can also use capture groups to search and replace using .replace()

Common flags

  • i : case insensitive
  • g : global (doesn't stop after first match)
  • m : multi-line

Common Anchors

  • ^ : start of string
  • $ : end of string
  • \A : Start of string (not affected by multi-line content)
  • \Z : End of string (not affected by multi-line content)

Most common quantifiers

  • {n} : Exactly n times
  • {n,} : At least n times
  • {m,n} : Between m and n times
  • ? : Zero or one time
  • + : One or more times
  • * : Zero, one or more times

Most common meta sequences

  • . : Any character but \n and \r
  • \w | \W : Any word character | Any non-word character
  • \d | \D : Any digit character | Any non-digit character
  • \s | \S : Any whitespace character | Any non-whitespace character

Character sets

  • [abc] : Will match either a, b or c
  • [1-9] : Will match any digit from 1 to 9
  • [a-zA-Z] : Will match any letter

Match any character but

  • [^abc] : Matches anything but a, b or c

Escape a character

  • \character (example : escaping + => \+)

Refer to a group (also used for capturing groups, look further):

  • (group of characters) (example : /(he)+/ will match 'hehehe'

One group or another

  • | : /^h((ello)|(ola))$/ will match both 'hello' and 'hola'
